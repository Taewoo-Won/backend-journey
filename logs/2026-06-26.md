# PART 3 — 핵심 판단을 기억으로 지킴 ★

25일의 가장 중요한 판단을, 정답 안 보고 지켰다:

- `balance < amount`(잔액 부족) → `InsufficientBalanceException` ✅ 바꿈

# PART 6 — 오늘 도출한 “왜?” 9문답 (개념 정리) ★★

복기하며 25일엔 안 보이던 “왜?“가 쏟아졌다. 이게 진짜 이해로 가는 문이다.

**1. 왜 `RuntimeException`을 extends 하나? (전에 만든 적 없는데)**
RuntimeException은 **자바가 이미 만들어 둔** 예외다. 내가 만드는 게 아니라 **물려받는(extends)** 것. 예외 기능(메시지 저장 등)은 공짜로 받고, **이름만 내 도메인 것으로** 단다. 상속 = 0부터 만들기가 아니라 “있는 것 위에 얹기”.

**2. 왜 `(String message)`를 받고 `super(message)`인가?**
이건 **생성자**. `new ...("계좌를 찾을 수 없습니다")`의 그 메시지를 받는 **입구**가 `(String message)`. 받은 메시지를 **부모(RuntimeException)에게 넘기는 게** `super(message)`. 부모가 메시지 저장 기능을 가졌으니 넘긴다 → 나중에 `e.getMessage()`로 꺼냄. (안 넘기면 null)

**3. 둘 다 RuntimeException이면 IllegalArgument/State 바꾼 의미 없지 않나?**
아니다. **부모가 같아도 “이름(타입)“이 다르다.** 바꾸기 전엔 둘 다 그냥 예외라 catch로 구분 불가. 바꾼 후엔 이름이 달라 `catch (AccountNotFoundException e)` / `catch (InsufficientBalanceException e)`로 **콕 집어 구분 가능.** RuntimeException은 “unchecked”라는 성질만 물려준 거고, **구분은 이름이 한다.**

**4. `(String message) {` 뒤의 `{`는 왜?**
그 `{`는 **생성자 몸통의 시작.** `( )`는 “뭘 받나”(입구), `{ }`는 “받아서 뭘 하나”(내용). 메서드·생성자·if·for 다 `{ }`로 내용을 감싼다.

**5. 왜 예외 클래스를 Account.java “밑에” 만드나?**
오해. **밑(하위)이 아니라 나란히(같은 위치)** 생긴 거다. Account 우클릭은 “같은 폴더에 새 파일” 만들려던 것뿐(src 폴더가 없어서). 5개 파일 다 같은 레벨에 평등.

**6. 왜 “잔액 부족”만 바꾸고 “0보다 커야”는 냅두나?** (25일 핵심)
성격이 다르다. “잔액 부족”=도메인 실패(비즈니스 사건)→커스텀. “0보다 커야”=잘못된 입력(사용자 실수)→JDK 예외가 이미 정확. **커스텀은 도메인에서 의미 있는 실패에만.** “충전 유도” 할 건 잔액 부족이지 음수 입력이 아니다.

**7. 왜 try 바로 안 가고 “— 이체 전 a 잔액” 출력을 먼저?** (제일 좋은 질문)
**그게 증명의 절반이다.** “돈 증발 없음”은 **이체 전후 비교**로 증명된다: 전 1100 → (실패) → 후 1100 = 안 변함 = 증발 없음. “이체 전”을 빼면 “후 1100”만 보고 원래 1100인지 줄었다 만 건지 **모른다.** before가 있어야 after가 의미를 가진다.

**8. “잡았다”만 치면 되지 왜 “잡았다: “ 콜론을?**
순수 **가독성용**, 기능 0. 라벨(“잡았다”)과 메시지를 콜론으로 구분해 읽기 쉽게. 빼도 똑같이 작동. 스타일 선택이지 문법 아님.

**9. 왜 “—이체 후”처럼 `---`를?**
8번과 동일, **보기용.** 출력 여러 줄 섞일 때 `---`가 중요한 줄을 눈에 띄게. 기능 0, 빼도 됨. 로그 읽기 쉽게 하는 습관(나중에 진짜 로그에서 유용).

> **제일 중요한 건 3·6·7.** 면접에서 “왜?“로 들어오는 지점이고, 오늘 정확히 그걸 스스로 물었다. 25일보다 깊어진 증거.

## 8. 개념 사전

- **복기(독타이핑):** 정답 안 보고 기억만으로 재현. 손가락에 새기는 훈련. 코드 완성보다 “직접 침”이 목적.
- **상속(extends):** 이미 있는 클래스(RuntimeException)의 기능을 물려받음. 0부터 안 만듦.
- **super(message):** 부모 생성자 호출. 메시지를 부모에 넘겨야 getMessage()로 꺼냄.
- **타입(이름)으로 구분:** 부모가 같아도 클래스 이름이 다르면 catch로 따로 잡음.
- **도메인 예외 절제:** 비즈니스 사건만 커스텀, 일반 입력 오류는 JDK 예외.
- **before/after 증명:** 상태 변화를 증명하려면 전과 후를 둘 다 찍어 비교.
- **Shift+F10:** IntelliJ 실행 단축키 = 컴파일+실행. javac+java를 한 키로.
- **git clone vs ZIP:** clone=GitHub 연결된 살아있는 복사본(push 가능). ZIP=죽은 복사본(연결 없음). **ZIP 금지.**
- **Compiler output:** .java를 번역한 .class를 저장할 폴더. 안 정하면 실행 에러.
