## 1. 잔디 5분 — mini-ledger IDE 부산물 정리

7/5에 적어둔 정리 항목을 오늘 털었다.

```bash
cd ~/mini-ledger
printf "out/\n.idea/\n*.iml\n" >> .gitignore
git add . && git commit -m "ignore IDE artifacts" && git push
```

판정:

```
[main 3480b61] ignore IDE artifacts
 1 file changed, 3 insertions(+)
   96f828d..3480b61  main -> main
```

**세 항목의 정체 (왜 추적에서 빼나):**

|항목      |정체                          |제외 이유                       |
|--------|----------------------------|----------------------------|
|`out/`  |IntelliJ가 만드는 **컴파일 산출물** 폴더|소스에서 언제든 재생성 — 결과물이지 원본이 아님 |
|`.idea/`|IntelliJ **프로젝트 설정** 폴더     |내 PC 환경에 종속 — 다른 환경에선 무의미/충돌|
|`*.iml` |IntelliJ **모듈 파일**          |위와 동일 — IDE 전용 메타데이터        |

공통 원칙: **repo에는 소스(원본)만. 도구가 만들어내는 것·환경에 종속된 것은 제외.** (server.log를 뺀 것과 같은 원칙의 연장이다 — 그건 “실행이 만들어내는 것”이었다.)

### ★ 오늘의 개념 한 톨 — 어제는 –cached가 필요했는데, 오늘은 왜 ignore만으로 끝났나

같은 .gitignore 작업인데 절차가 달랐다:

- **어제(7/7) server.log:** `.gitignore` 추가 + **`git rm --cached`** 두 단계 필요 — 이미 커밋돼 **추적 중**이었기 때문. ignore는 추적 중인 파일엔 무력하다.
- **오늘 out/·.idea/·*.iml:** `.gitignore` 추가 **한 단계로 끝** — 이 항목들은 mini-ledger repo에 **커밋된 적이 없다**(push를 VPS에서만 했고, IDE 부산물은 사지방 PC 로컬에만 생겼으므로). 추적 목록에 없는 파일은 ignore가 바로 먹는다.

**갈림 기준 한 줄: “그 파일이 이미 추적 중이냐.”** 추적 중이면 2단계(ignore + rm –cached), 아니면 1단계(ignore만). 판정이 필요하면 `git ls-files | grep 파일명`으로 추적 여부를 직접 확인할 수 있다. — 같은 도구를 이틀 연속 다른 상황에서 쓰면서, 어제의 개념(–cached의 존재 이유)이 오히려 단단해졌다.

(부수 확인: 이 커밋은 7/5 이메일 통일 이후 mini-ledger에서 찍은 **첫 커밋** — noreply 체계에서 잔디가 정상적으로 이어지는 것도 겸사겸사 확인된 셈이다.)

## 2. 오늘의 핵심 판단 — 복기 방식의 전환

**원래 예약(7/8~9):** 빈 화면 재현 — 원자성 트레이스(a=1010 버그 버전) + checkBalance를 기억만으로 재작성.

**전환한 것:** 오늘 하루는 **인쇄용 심화 문답집** 방식으로. 7/6~7/7 전 과정을 질문·모범답안(Q&A)으로 문서화하고, A4로 인쇄해 정독·셀프테스트한다.

**전환 이유:** 복기 대상이 달랐다 — 예약돼 있던 재현 과제는 7/5분(원자성·Map)이었는데, 지금 굳혀야 할 것은 **7/6~7/7의 스프링 전환**(record, @RequestBody, 상태를 가진 서버 등 첫 등장 개념 덩어리)이다. 전환기 개념은 “손으로 재현”보다 먼저 **구조화된 설명을 정독하며 이해를 재구축**하는 단계가 맞다고 판단했다. 인쇄물은 화면 없이(생활관에서도) 반복 가능하다는 실용 이점도 있다.

**두 방식의 성격 — 상보적이다:**

|      |빈 화면 재현              |문답집 정독/셀프테스트         |
|------|---------------------|---------------------|
|훈련 방향 |**출력**(꺼내기) — 손이 기억하나|**입력+구조화** — 이해가 서 있나|
|잘 잡는 것|문법·코드의 근육 기억         |개념 사이의 연결·“왜”        |
|실패 신호 |손이 멈추는 지점            |답을 읽어도 갸웃한 지점        |

그래서 **버린 게 아니라 연결했다:** 문답집 **부록에 “빈 화면 재현 체크리스트 8개”**를 넣어, 정독이 끝나면 인쇄물을 덮고 재현으로 넘어가게 설계했다. 그리고 7/5분 재현(원자성 트레이스 + checkBalance)은 **예약 그대로 유지** — 문답집 말미에 다음 복기 후보로 명시해뒀다.

## 6. 회고

19일 만에 처음 감속했다. 어색했지만 — 원칙을 박아둔 이유가 정확히 이런 날을 위해서였다. “마일스톤을 닫은 다음 날이 숨 고르는 자리”라는 문장은 관문 조기 충족이라는 숫자가 있어야 힘을 갖고, 그 숫자는 6월부터의 연속 가동이 만들었다. 쉼이 계획의 일부가 되려면 그 앞의 달림이 계획대로였어야 한다는 것 — 오늘 그게 맞물렸다.

실질 산출물은 문답집이다. 복기 방식을 스스로 설계 변경했다는 점이 코드 한 줄보다 남는다 — “어떻게 공부할 것인가” 자체를 도구로 만들었다. 전환기 개념 덩어리(record, 잭슨, 상태를 가진 서버)는 손 재현보다 구조화된 정독이 먼저라고 판단했고, 대신 재현을 버리지 않고 부록 체크리스트로 이어 붙였다. 인쇄물 한 부로 정독→덮고 재현까지 한 사이클이 되게 — 학습 파이프라인을 짠 셈이다.

그리고 작지만 오늘의 개념 한 톨: 같은 .gitignore인데 어제는 두 단계, 오늘은 한 단계였다. “이미 추적 중이냐”라는 갈림 기준은 어제 –cached를 배울 때보다 오늘 그것이 **불필요한 경우**를 만나면서 오히려 선명해졌다. 개념은 쓰일 때가 아니라 안 쓰일 때를 구분할 수 있을 때 내 것이 된다.

내일부터의 길은 명확하다. 재진입점(예외→HTTP 번역)은 어제 위치를 찍어뒀고, 진입 비용까지 낮다(a=60, curl 한 방이면 500이 관찰된다). 오늘 고른 숨이 그 진입을 가볍게 만들 것이다.
