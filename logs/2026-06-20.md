## Phase 3 — 증분 1: Account 클래스 작성 ★오늘의 핵심★

### 레슨 0 — 변수 (값을 담는 상자)
```java
long balance = 1000;
balance = balance + 500;    // → 1500
System.out.println(balance);
```
- `long balance = 1000;` = "`balance`라는 상자(`long`=정수)에 1000 넣기"
- `balance = balance + 500;` = "상자 값을 (현재값 + 500)으로 바꿔 다시 넣기"
- → deposit의 `balance = balance + amount`가 *이것과 똑같음*을 이해. (어제 안 읽히던 그 줄)

### Account 구조 이해
- `public class Account { }` = 계좌 *설계도(클래스)*
- 필드 `id`(고유번호) · `owner`(주인) · `balance`(잔액)
- `private` = 캡슐화(밖에서 직접 못 건드림), `final` = 재할당 불가
- 생성자 = 계좌 만들 때 필드 채움
- `getBalance()` = 잔액 읽기 통로
- `deposit`/`withdraw` = 내가 채울 두 메서드

### deposit 구현 — 직접 작성
```java
public void deposit(long amount) {
    if (amount <= 0) {
        throw new IllegalArgumentException("입금액은 0보다 커야 합니다");
    }
    balance = balance + amount;
}
```
한 줄씩:
- `if (amount <= 0) { ... }` → 0 이하 금액 막기 (나쁜 입력 차단)
- `throw new IllegalArgumentException(...)` → 에러 던지고 메서드 멈춤
- `balance = balance + amount;` → 잔액에 더해 다시 저장

### withdraw 구현 — 직접 시도 → 교정

**처음 시도에서 잡은 실수 3개 (← 진짜 배운 부분):**
1. 메서드 이름을 또 `deposit`으로 씀 → *같은 이름 중복 금지*, 빼기니까 `withdraw`
2. **설명을 코드에 붙여넣음** — `"상태가 안 됨" = 잔액 부족에 딱` 은 `IllegalStateException`이 뭔지 *설명한 말*이지 코드가 아님. (교훈: **설명 ≠ 코드**)
3. `amount <= 0` 검사를 빠뜨림 — *두 검사 다* 필요

**완성:**
```java
public void withdraw(long amount) {
    if (amount <= 0) {
        throw new IllegalArgumentException("출금액은 0보다 커야 합니다");
    }
    if (balance < amount) {
        throw new IllegalStateException("잔액이 부족합니다");
    }
    balance = balance - amount;
}
```
한 줄씩:
- 검사 1: `amount <= 0` → deposit과 동일 (음수 출금 막기)
- 검사 2: `balance < amount` → 잔액 부족이면 막기 (`IllegalStateException` = "상태가 안 됨")
- `balance = balance - amount;` → 통과 후 빼기 (deposit은 `+`, 이건 `-`)

**핵심 패턴:** *검사 두 개 먼저 → 통과 후에야 balance 변경.* = 과제의 **"바꾸기 전에 검증"** 원칙. 잘못된 출금이면 *빼기 전에* 막아 잔액 보호 → 나중의 ACID·트랜잭션 사고의 씨앗.

### 적용한 결제 엔지니어 원칙
- **돈은 `long`** (double 금지 — 부동소수점 = 돈 버그)
- **검증 먼저** (상태 바꾸기 전 검사)
- **도메인 실패 = 예외** (그냥 진행 X)

### 파일 생성 + 컴파일 (여기서 환경 개념도 배움)
- **셸 vs 에디터 구분:** `cat`·`javac`·`git`은 *터미널 명령*, nano *안*이 아님. (cat 명령을 nano에 넣으려다 교정.)
- **디렉토리:** `~`=홈, 프로젝트는 `~/mini-ledger` 폴더. `cd`로 이동. (프롬프트 `~/mini-ledger#`로 바뀌는 걸로 확인)
- **cat heredoc으로 파일 통째 생성:**
  ```bash
  mkdir -p ~/mini-ledger && cd ~/mini-ledger
  cat > Account.java << 'EOF'
  ... (전체 코드) ...
  EOF
  ```
  ( `EOF` ~ `EOF` 사이를 Account.java 내용으로 *통째 덮어씀.* 중복·위치 걱정 없이 한 방.)
- **컴파일:** `javac Account.java` → `Account.class` 생성 = *에러 0, 컴파일 성공.* (`ls`로 `Account.class` 확인)

### 완성된 Account.java (전체)
```java
public class Account {
    private final String id;
    private final String owner;
    private long balance;

    public Account(String id, String owner, long initialBalance) {
        this.id = id;
        this.owner = owner;
        this.balance = initialBalance;
    }

    public long getBalance() {
        return balance;
    }

    public void deposit(long amount) {
        if (amount <= 0) {
            throw new IllegalArgumentException("입금액은 0보다 커야 합니다");
        }
        balance = balance + amount;
    }

    public void withdraw(long amount) {
        if (amount <= 0) {
            throw new IllegalArgumentException("출금액은 0보다 커야 합니다");
        }
        if (balance < amount) {
            throw new IllegalStateException("잔액이 부족합니다");
        }
        balance = balance - amount;
    }
}
```
