## Phase 1 — Java 환경 구축

**왜:** JDK 21 LTS(Temurin) 채택 — 한국 핀테크 프로덕션 주류(17~21), 채용 요건과 정렬, 튜토리얼·라이브러리 호환 최대.
**도구 선택:** `apt` 대신 **SDKMAN** — JDK 여러 버전 전환 + Gradle·Kotlin까지 한 도구로 관리. 로드맵(Java→Kotlin, Gradle)에 정확히 맞음.

**한 일:**

1. SDKMAN 의존성 + 설치
   ```bash
   sudo apt install -y zip unzip curl
   curl -s "https://get.sdkman.io" | bash
   source "$HOME/.sdkman/bin/sdkman-init.sh"
   sdk version
   ```
2. JDK 21 (Temurin) 설치
   ```bash
   sdk list java        # 페이저에서 /tem 검색 → 21.0.11-tem 찾음
   sdk install java 21.0.11-tem
   java -version         # 21.0.11 확인
   javac -version        # 컴파일러도 확인
   ```
3. 첫 Java 프로그램
   ```bash
   mkdir -p ~/java-practice && cd ~/java-practice
   nano Hello.java
   ```
   ```java
   public class Hello {
       public static void main(String[] args) {
           System.out.println("Hello from VPS!");
       }
   }
   ```
   ```bash
   javac Hello.java && java Hello    # → "Hello from VPS!"
   ```

**배운 것:**
- SDKMAN으로 JDK 버전 관리 (`sdk list` / `sdk install`)
- **클래스명 = 파일명** 규칙 (`Hello.java` ↔ `public class Hello`)
- **편집 → 컴파일 → 실행** 루프 (`nano` → `javac` → `java`)
- 컴파일(`javac`, `.class` 생성)과 실행(`java`)은 별개 단계
- 폰 작업 팁: Termius **extra keys**(Ctrl), nano 저장 `Ctrl+O`/종료 `Ctrl+X`, **↑로 직전 명령 재호출**

## Phase 2 — Git + GitHub 연동

**왜:** 버전관리 + *공개 잔디*(평가관이 보고, "일찍 시작" 효과). 코드를 GitHub에 쌓는다.
**인증 방식 결정:** **SSH 키(ed25519)** — 만료 관리 불필요(매일 커밋에 유리), AWS EC2 등 서버 인증에 그대로 전이되는 기초 스킬.

**한 일:**

1. git 설정
   ```bash
   git config --global user.name "Taewoo-Won"
   git config --global user.email "288912277+Taewoo-Won@users.noreply.github.com"
   git config --global init.defaultBranch main
   git config --global --list        # 확인
   ```
   - 이메일은 **GitHub noreply** 주소 사용 → (a) 개인 이메일 비공개 (b) 잔디 확실히 찍힘
   - *주의했던 점:* 처음에 예시 플레이스홀더 `"표시할이름"`을 그대로 넣었다가 실제 값으로 교정

2. SSH 키 생성 (+ 보안 점검 차 1회 재생성)
   ```bash
   ssh-keygen -t ed25519 -C "github-vps"    # passphrase 빈 값
   ```
   - 중간에 "내 키가 노출된 거 아닌가" 점검 → *공개키는 원래 공개용*임을 확인, 안심 차원에서 키 재생성:
     ```bash
     rm ~/.ssh/id_ed25519 ~/.ssh/id_ed25519.pub
     ssh-keygen -t ed25519 -C "github-vps"
     ```

3. 공개키 GitHub 등록
   ```bash
   cat ~/.ssh/id_ed25519.pub          # ssh-ed25519 ... 한 줄
   ```
   → GitHub 웹 → Settings → SSH and GPG keys → New SSH key → **Authentication Key**로 추가

4. 연결 테스트
   ```bash
   ssh-keyscan -t ed25519 github.com >> ~/.ssh/known_hosts   # 호스트 키 등록
   ssh -T git@github.com               # → "Hi Taewoo-Won! ..."
   ```
   - GitHub 호스트 키 지문(`SHA256:+DiY3wvvV6...`)이 GitHub 공식 값과 *일치*하는지 검증 후 신뢰

5. 로컬 repo + 첫 커밋
   ```bash
   echo "*.class" > .gitignore         # 빌드 산물(.class)은 커밋 제외
   git init
   git add .
   git status                          # .gitignore, Hello.java staged / Hello.class 제외 확인
   git commit -m "first commit: hello world on VPS"   # → c6db593
   ```

6. GitHub repo 생성 + push
   - GitHub 웹 → New repository → 이름 `java-practice` → README·.gitignore·license **전부 해제**(빈 repo, 안 그러면 push 충돌)
   ```bash
   git remote add origin git@github.com:Taewoo-Won/java-practice.git
   git push -u origin main             # → [new branch] main -> main
   ```
   - *겪은 일:* repo를 먼저 안 만들고 push해서 `Repository not found` → repo 생성 후 재push 성공

7. 검증 (완료 기준)
   - repo에 `Hello.java` 보임 ✓
   - 프로필 잔디 초록 1칸 ✓ (repo가 **Public**이라 표시됨 — Private이면 공개 프로필에 안 뜸)

**배운 것:**
- **공개키 vs 개인키** = 자물쇠(공개) vs 열쇠(비밀). 노출돼서 문제 되는 건 *개인키*뿐
- **HTTPS vs SSH** = GitHub 들어가는 두 개의 문. HTTPS는 비번/토큰, SSH는 키
- **remote / origin** = 로컬 git이 저장해둔 "GitHub 주소 별명" (연락처 저장처럼)
- **호스트 키 지문 검증** = `yes` 치기 전 중간자 공격 방어
- `.gitignore` = 빌드 산물 등 커밋 제외 / `git init`·`add`·`commit`·`push`의 역할

## Phase 3 — gh CLI 설치 (보너스)

**왜:** 다음 repo부터 Safari 웹 폼 없이 *터미널 한 줄*로 생성. 폰 워크플로 마찰 최소화.

**한 일:**
```bash
sudo apt install gh -y               # gh 2.45.0
gh auth login
# → GitHub.com / SSH / Skip(키 이미 등록) / Login with web browser
# → device flow: 코드(9842-410C) → Safari github.com/login/device → Authorize
gh auth status                       # → Logged in as Taewoo-Won
```

**배운 것:**
- **gh ≠ git.** git = 버전관리(add/commit/push, *매일*). gh = GitHub 관리(repo 생성·PR·이슈, 원래 웹에서 하던 것)
- **device flow** = 터미널에 비번 안 치고 일회용 코드로 브라우저서 인증
- 향후 새 repo = `gh repo create 이름 --public --source=. --remote=origin --push` *한 줄* (생성+연결+push)
