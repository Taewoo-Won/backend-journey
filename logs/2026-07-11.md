## 1. 오늘의 판단 — 꼼수 커밋 거부

시간이 없어 잔디만 유지하려던 상황. **처음 제안된 방식:**

```bash
echo "$(date +%F): 잔디 유지 — 내일 재개" >> study-log.txt
git commit -m "keep streak: defer exception handler to 7/12"
```

**이걸 거부했다. 이유 둘:**

1. **커밋 메시지는 영구 이력이다.** GitHub의 커밋 로그는 채용담당자가 볼 수 있는 공개 기록이고, `keep streak` / `defer` 같은 메시지는 “이날 나는 미뤘고 잔디용으로 때웠다”를 **스스로 문서화**하는 꼴이다. 이력서에 “이날은 놀았음”이라 적는 것과 같다.
1. **메시지만 고치는 것도 반쪽이다.** 메시지를 그럴듯하게 바꿔도, `study-log.txt`에 날짜 한 줄 추가하는 커밋은 들여다보면 **내용이 빈 커밋**이다. 포장만 바꾼 것.

**대안: 1분이 걸리더라도 실질 변경을 만든다.** 마침 mini-ledger의 README가 **딱 1줄**(제목만)인 껍데기였다 — 졸업작 repo인데 설명이 없는 상태. 이걸 채우는 건 꼼수가 아니라 정상 작업이다.

```bash
cd ~/mini-ledger && wc -l README.md      # → 1  (껍데기 확인)

cat >> README.md << 'EOF'                # ★ >> (추가) — 기존 1줄 보존, 덮어쓰기 아님

순수 Java로 바닥부터 만든 결제 원장 학습 프로젝트. 프레임워크 없이 결제 시스템의 핵심 규칙을 직접 구현했다.

## 기능
- 계좌 생성 / 이체 / 환불 (수수료 정산 포함)
- **멱등성** — 동일 idempotencyKey 중복 처리 방지
- **원자성 검증** — 상태 변경 전 사전 검증 (반쪽 커밋 방지)
- **복식부기 검증** — 매 이체마다 전체 잔액 합 = 0 확인
- 거래 조회 (계좌별 / 금액 조건별)

## 이후
이 도메인을 그대로 이식해 Spring Boot HTTP API로 확장 → https://github.com/Taewoo-Won/ledger-api
EOF
git add README.md && git commit -m "add README: project overview and core guarantees" && git push
```

판정:

```
[main 10d37db] add README: project overview and core guarantees
 1 file changed, 15 insertions(+)
   b4ac357..10d37db  main -> main
```

**결과:** 커밋 메시지 떳떳(`add README`) + 내용 실질(첫인상 파일 완성) + 잔디 유지. 세 개를 다 얻었다.

**★ 문법 메모 — `>` vs `>>`:** `cat > 파일`은 **덮어쓰기**(기존 내용 소멸), `cat >> 파일`은 **뒤에 추가**. 기존 1줄(제목)을 살려야 했으므로 `>>`가 정답이었다. 하나 틀렸으면 제목이 날아갔다.

## 2. 원칙으로 남길 것

> **잔디는 목적이 아니라 결과다.** 커밋이 없는 날은 커밋을 짜내는 게 아니라, **실질 변경이 있는 작업을 1분이라도 하는 것**이다. 그것도 없으면 **잔디를 비우는 게 정답**이다 — 20일 연속이 21일 되는 것보다 이력이 정직한 게 값어치 있다.

## 5. 회고

오늘의 실질 산출물은 README 15줄이지만, 남는 건 그 앞의 판단이다. 잔디를 위해 빈 커밋을 만들자는 제안을 받고 **그걸 거부한 게 오늘의 전부**다.

이유가 스스로 명확했다. 커밋 로그는 내가 보려고 쓰는 일기가 아니라 **남이 볼 수 있는 이력**이다. `defer`, `keep streak` 같은 단어를 남기는 순간, 그 저장소는 “매일 뭔가 한 사람”이 아니라 “매일 뭔가 한 척한 사람”의 기록이 된다. 잔디 하나 지키자고 이력의 신뢰를 파는 건 나쁜 거래다.

그리고 메시지만 바꾸는 우회도 거부했다 — 그건 문제를 이름만 바꿔 덮는 것이다. 대신 진짜 필요한 작업(비어 있던 README)을 찾아 1분에 끝냈다. 필요한 일이 있어서 다행이었지, 없었다면 잔디를 비우는 게 맞았다.

> **한 줄:** 빈 커밋을 거부하고 실질(README)로 대체 — 잔디는 목적이 아니라 결과다.
