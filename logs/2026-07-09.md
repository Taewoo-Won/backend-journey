## 1. 재현 훈련 1호 — 실행과 판정

tmux 세션 안에서, 아무것도 안 보고:

```bash
nano ~/test-recall.java     # HealthController를 기억만으로 재작성
```

**1차 결과 — 뼈대 전부 나옴, 교정 2곳:**

```java
package com.ledger.api        // ✗ 세미콜론 누락 — 선언문은 ; 필수
...
@RestController {             // ✗ 애노테이션에 중괄호 — 스티커는 블록이 아니다
public class HealthController {
```

- **나온 것:** import 2줄(GetMapping·RestController), `@GetMapping("/health")`, 메서드 시그니처, `return "ledger-api alive";` — 3일 전 처음 본 코드의 골격이 기억에서 전부 나왔다.
- **교정 ①:** `package com.ledger.api;` — 문장(선언문)이므로 세미콜론으로 끝나야 한다.
- **교정 ②:** 애노테이션은 **단독 줄로 끝난다.** `@RestController` 다음 줄에 `public class ... {`. 중괄호를 붙이면 여는 괄호가 하나 남아 컴파일 자체가 안 된다.

**2차 재작성 — 통과.** 괄호 짝(class 열고 → 메서드 열고 닫고 → class 닫고)까지 정상. 컴파일 가능한 형태 확인.

**판정:** 재현 통과. 복기 빚에 대한 걱정은 절반은 과장이었다 — 골격은 붙어 있고, 헐거운 건 문법 마감 디테일이다.

## 2. tmux — 오늘부터의 작업 방식

**한 줄 정의: SSH가 끊겨도 죽지 않는 터미널 세션.**

```bash
tmux new -s dev        # 세션 생성 (최초 1회)
tmux attach -t dev     # 끊긴 뒤 재접속 — 하던 화면 그대로 복귀
# Ctrl+b 누른 뒤 %  → 좌우 분할 (한쪽 서버 로그, 한쪽 curl)
```

- 폰 작업의 두 고통(연결 끊기면 전멸 · 창 하나)을 동시에 해결. `> server.log 2>&1 &` 곡예의 상당 부분을 대체한다.
- 사지방에서도 `tmux attach` — **폰에서 하던 화면을 큰 모니터로 이어받는다.** 두 기기가 한 작업대가 된다.

**환경 판단 확정:** 노트북 반입 트랙 폐기 — 부대 내 핫스팟 불가이므로 반입해도 인터넷 없는 개발환경 = 비용 대비 무의미. 남는 질의는 **블루투스 키보드** 반입 가능 여부 하나. IntelliJ 성능 튜닝 과제도 폐기(초기화되는 공용 저사양 PC에 투자 안 함) — 사지방 PC는 SSH 단말/열람용으로 역할 재정의.

## 3. 애노테이션 3대 질문 — 발제와 답 요지 (★소화 미완, 재독용)

재현 중 나온 진짜 질문 셋. 답의 관통선 하나: **달라진 건 자바가 아니라 호출자다.** Main 시절엔 내가 직접 불렀으니 아무한테 알릴 게 없었고, 지금은 스프링이 대신 부르므로 “누가 HTTP 담당인지, 어느 URL이 어느 메서드인지”를 스프링에게 알려야 한다 — 그 언어가 `@`다.

**Q1. import로 똑같이 땡겨왔는데 왜 얘들만 @를 붙이나?**
→ import가 가져온 이름의 **종족**이 다르다. `Map`은 클래스(재료)라 `new` 해서 쓰고, `RestController`는 **애노테이션(스티커)**이라 붙이는 것만 된다. @가 “붙인다”는 문법이다. (mini-ledger에 @가 없던 이유 = 스티커를 읽을 주체(스프링)가 없었다.)

**Q2. 왜 @RestController와 @GetMapping이 “두 층”인가?**
→ @GetMapping이 @RestController “안에” 있는 게 아니라, **클래스 안에 메서드가 있는 것**(Bank 안에 transfer 있던 그 구조)이고 스티커가 각자 자기 층에 붙은 것. 역할이 다르다: 클래스 스티커 = 스캔이 찾는 **등록 단위**(“이 건물은 HTTP 부서”), 메서드 스티커 = 그 안의 **개별 창구**(“이 창구는 /health 업무”). 한 클래스가 창구 여러 개를 가질 수 있으므로 URL은 메서드 단위가 맞다.

**Q3. `("/health")`는 왜 넣나?**
→ 애노테이션은 값을 받을 수 있다 — **스티커에 쓰는 글자.** @GetMapping만으로는 “GET을 받는다”까지고, `"/health"`가 **주소판**이다. 부팅 때 URL 표에 `GET /health → health()`로 등록되고, 요청이 오면 표에서 찾아 호출한다. (7/7의 `@PostMapping("/transfer")`가 같은 패턴.)

**★ 미완 표시:** 시간 부족으로 이 답들을 소화 못 하고 세션 종료. **다음 세션 첫 5분 = 이 §3 재독**부터. (문답집 Q10~Q13이 같은 내용의 확장판이다.)
