## 오늘 복기한 내용

어제는 LCS 알고리즘 구현에 집중해서 넘어갔지만, 오늘 다시 읽어보니 더 큰
변화는 알고리즘이 아니라 채점 결과를 표현하는 방식이었다.

기존 방식은 일치율 하나만 보여줬다.

``` text
일치율 70%
```

하지만 숫자 하나만으로는 왜 틀렸는지 알 수 없었다.

그래서 결과를 다음처럼 분리했다.

-   재현율
-   빠뜨림
-   군더더기

이렇게 나누면 같은 점수라도 원인을 바로 확인할 수 있다.

예를 들어,

``` text
재현율 92%
빠뜨림 1
군더더기 0
```

이라면 기억하지 못한 줄이 하나라는 뜻이고,

``` text
재현율 92%
빠뜨림 1
군더더기 1
```

이라면 기억하지 못한 줄과 잘못 추가한 줄이 동시에 있다는 뜻이다.
